# Dictionnaire des champs XDM


[!DNL Experience Data Model] (XDM) définit un ensemble de champs standard pouvant être utilisés lors de la composition de schémas Adobe Experience Platform. L’utilisation de ces champs entraîne un comportement cohérent dans les [!DNL Platform] , car tous les services qui utilisent XDM comprendront explicitement les définitions spécifiques fournies par ces champs.

Voir [principes de base de la composition des schémas](composition.md) pour plus d’informations sur les blocs de création des schémas, y compris les principes et les bonnes pratiques de définition des schémas pour [!DNL Experience Platform].

Le tableau ci-dessous inclut le nom du champ (tous les espaces de noms des champs XDM ne sont pas définis à l’aide de &quot;xdm&quot;), ainsi que son titre et sa description, son type et son format, le cas échéant, ainsi que le type XDM correspondant et les ressources (schémas, groupes de champs de schéma et types de données) dans lesquels le champ peut être trouvé.

Vous pouvez utiliser la fonctionnalité de recherche dans le [!DNL Schema Editor] pour afficher les détails de ces ressources, comme indiqué dans la section [Tutoriel de l’éditeur de schémas](../tutorials/create-schema-ui.md#browse).

>[!NOTE]
>
>Si un champ est marqué « **OBSOLÈTE** », cela signifie qu’il ne doit plus être utilisé dans les nouvelles implémentations. Le champ reste disponible pour une compatibilité descendante, mais ne sera plus pris en charge dans les prochaines versions.
