# Dictionnaire de champ Modèle de données d’expérience (XDM)


Le modèle de données d’expérience (XDM) définit un ensemble de champs standard qui peuvent être utilisés lors de la composition du Adobe Experience Platform. L’utilisation de ces champs entraîne un comportement cohérent entre les services de plateforme, car tous les services qui utilisent XDM comprendront explicitement les définitions spécifiques fournies par ces champs.

Voir les [bases de la composition](composition.md) des  pour plus d’informations sur les éléments constitutifs des  de, y compris les principes et les meilleures pratiques pour la définition de l’ pour la plateforme d’expérience.

Le tableau ci-dessous inclut le nom du champ (tous les champs XDM ne sont pas des espaces de noms utilisant &quot;xdm&quot;), ainsi que son titre et sa description, son type et son format, le cas échéant, ainsi que le type XDM correspondant et les ressources (mixins, types de données et ) dans lesquelles le champ est accessible.

Vous pouvez utiliser la fonctionnalité de recherche dans l’éditeur de  de pour  détails de ces ressources, comme illustré dans le didacticiel de l’éditeur de  de l’éditeur de contenu.

>[!NOTE] Si un champ est marqué &quot;**DÉCONSEILLÉ**&quot;, cela signifie qu’il ne doit plus être utilisé dans les nouvelles implémentations. Le champ reste disponible pour une compatibilité descendante, mais ne sera plus pris en charge dans les versions ultérieures.
