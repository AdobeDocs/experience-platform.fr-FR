## Mises à jour de l’interface utilisateur {#ux}

Adobe Experience Platform publie des mises à jour sur la barre de domaine et d&#39;en-tête afin d&#39;améliorer votre expérience et de vous unir à d&#39;autres applications [!DNL Experience Cloud].

- Il est plus facile de basculer entre vos organisations ou vers une autre application
- Amélioration de l’aide à l’utilisateur, y compris les articles présentés et la documentation contextuelle dans le menu Aide
- Capacité à fournir des commentaires sur [!DNL Experience Platform] et les tickets d&#39;assistance

Le déploiement de la nouvelle expérience est graduel. Vous pouvez vue l’expérience à l’adresse [https://experience.adobe.com/platform](https://experience.adobe.com/platform).

## [!DNL Data Science Workspace] {#dsw}

[!DNL Data Science Workspace] utilise l&#39;apprentissage automatique et l&#39;intelligence artificielle pour libérer des informations de vos données. Intégré dans Adobe Experience Platform, [!DNL Data Science Workspace] vous aide à faire des prédictions à l’aide de vos contenus et de vos ressources de données dans les solutions d’Adobe. [!DNL Data Science Workspace] permet d&#39;y parvenir en utilisant [!DNL JupyterLab]. [!DNL JupyterLab] est une interface utilisateur web pour [[!DNL Project Jupyter]](https://jupyter.org/) et est étroitement intégré à Adobe Experience Platform. Il fournit un environnement de développement interactif pour que les chercheurs en données puissent travailler avec des blocs-notes, du code et des données [!DNL Jupyter].

**Nouvelles fonctionnalités**

| Fonctionnalité | Description |
|--- | ---|
| [!DNL JupyterLab Launcher] | [!DNL JupyterLab Launcher] comprend désormais des démarreurs pour les blocs-notes [!DNL Spark] 2.4. [!DNL Spark] Les démarreurs de bloc-notes 2.3 sont maintenant marqués comme obsolètes et sont définis pour être supprimés dans une version ultérieure. |
| [!DNL Spark] 2,4 | Les nouvelles recettes Scala ([!DNL Spark]) et PySpark utilisent désormais [!DNL Spark] 2.4. |
| Noyaux | Les blocs-notes Scala ([!DNL Spark]) sont maintenant créés via le noyau Scala. Les blocs-notes PySpark sont maintenant créés via le noyau [!DNL Python]. Le noyau [!DNL Spark] et PySpark sont déconseillés et sont définis pour être supprimés dans une version ultérieure. |
| Recettes | Les nouvelles recettes PySpark et [!DNL Spark] suivent maintenant le flux de production [!DNL Docker] semblable aux recettes [!DNL Python] et R. |

Pour plus d&#39;informations sur la migration de vos blocs-notes et de vos recettes pour utiliser [!DNL Spark] 2.4, consultez le [guide d&#39;accès aux données des blocs-notes](../../data-science-workspace/jupyterlab/access-notebook-data.md). Pour plus d’informations sur Data Science Workspace, voir la [documentation d’aperçu](../../data-science-workspace/home.md).

## [!DNL Destinations] {#destinations}

Dans [Plate-forme de données client en temps réel](../../rtcdp/overview.md), les destinations sont des intégrations préétablies avec les plateformes de destination qui activent les données à ces partenaires de manière transparente.

**[!DNL Facebook]**

Le CDP en temps réel prend désormais en charge l’activation des données à [!DNL Facebook], ce qui vous permet d’activer des profils pour vos campagnes [!DNL Facebook] pour le ciblage, la personnalisation et la suppression des audiences en fonction des courriers électroniques hachés.

Pour plus d’informations sur la nouvelle fonctionnalité, voir la page [destination Facebook](../../destinations/catalog/social/facebook.md).

**[!DNL Amazon Kinesis]et les destinations  [!DNL Azure Event Hubs] d’enregistrement cloud en flux continu**

Le CDP en temps réel prend désormais en charge l’activation des données vers les destinations d’enregistrement cloud en flux continu, ce qui vous permet d’exporter des données d’audience et des événements vers ces destinations au format JSON. Vous pouvez ensuite décrire la logique métier au-dessus de ces événements dans vos destinations. Voir ci-dessous pour plus de détails :

>[!NOTE]
>
>Les destinations [!DNL Amazon Kinesis] et [!DNL Azure Event Hubs] dans le CDP en temps réel sont actuellement en version bêta. La documentation et les fonctionnalités peuvent changer.

| [(bêta) Destination Amazon](../../destinations/catalog/cloud-storage/amazon-kinesis.md) | Cet article explique comment créer une connexion sortante en temps réel à votre [!DNL Amazon Kinesis] enregistrement de diffusion de données à partir de Adobe Experience Platform. |
| [(bêta) Destination des centres de Événement Azure](../../destinations/catalog/cloud-storage/azure-event-hubs.md) | Cet article explique comment créer une connexion sortante en temps réel à votre [!DNL Azure Event Hubs] enregistrement de diffusion de données à partir de Adobe Experience Platform. |

## [!DNL Experience Platform Web SDK] et [!DNL Experience Platform Edge Network] {#edge}

Les [!DNL Experience Platform Web SDK] et [!DNL Experience Platform Edge Network] permettent aux utilisateurs d&#39;envoyer des données à Adobe Experience Platform et à d&#39;autres solutions d&#39;Adobe en temps réel pour les périphériques et navigateurs des utilisateurs finaux. La liste la plus récente des cas d&#39;utilisation se trouve dans notre [feuille de route publique](https://github.com/adobe/alloy/projects/5) qui est fréquemment mise à jour.

**Nouvelles fonctionnalités**

| Fonctionnalité | Description |
|--- | ---|
| Prise en charge de l&#39;ECID | Le SDK prend en charge l’ECID prêt à l’emploi sans bibliothèques ni informations supplémentaires à installer. |
| Interface utilisateur de configuration | Gérez vos paramètres d&#39;ID de configuration à l&#39;aide de la nouvelle interface utilisateur de configuration Edge dans [!DNL Launch]. Vous devez être sur la liste autorisée pour pouvoir accéder à cette interface. |
| Groupe de champs du schéma SDK Web Adobe Experience Platform | Groupe de champs à utiliser avec [!DNL Experience Platform web SDK] qui englobe tous les champs pris en charge. |
| Contrôles du consentement du cours | Permet aux sociétés de contrôler l’inclusion et l’exclusion de [!DNL Experience Platform Web SDK]. |
| Prise en charge du débogage côté client dans le nouveau [!DNL Experience Cloud Debugger Extension] | Voir les requêtes de [!DNL Experience Platform web SDK] ainsi que les traces de bord pour voir comment les données circulent dans le système. |
| Adobe Analytics | Envoyez des données aux suites de rapports [!DNL Analytics] par le biais de la configuration du bord. XDM est aplati dans les données contextuelles et prend en charge le balisage multi-suite. |
| Adobe Target | Soutien à Adobe Target. Comprend le compositeur d’expérience visuelle, le compositeur d’après les formulaires, A/B, XT, Automated Personalization, MVT. |
| Prise en charge de Adobe Audience Manager | Prise en charge des synchronisations d’ID [!DNL Audience Manager], des destinations URL et des destinations de cookies. |
| Synchronisation des identités | Renommé `setCustomersIds` en `syncIdentity` pour le rendre plus clair. |
| Générateur d’objets XDM | Dans l’extension de lancement, vous pouvez désormais créer des objets XDM en tant qu’éléments de données. |

Pour plus d&#39;informations sur [!DNL Platform Web SDK] et [!DNL Edge Network], consultez la [documentation](../../edge/home.md).

## [!DNL Real-time Customer Profile] {#profile}

Adobe Experience Platform vous permet d’offrir aux clients des expériences coordonnées, cohérentes et pertinentes, quel que soit l’endroit ou le moment où ils interagissent avec votre marque. [!DNL Real-time Customer Profile] permet de visualiser une vue holistique de chaque client qui combine des données provenant de plusieurs canaux, y compris des données en ligne, hors ligne, CRM et tierces. [!DNL Profile] vous permet de consolider vos données client disparates en une vue unifiée offrant un compte d’activité horodaté de chaque interaction client.

**Nouvelles fonctionnalités**

| Fonctionnalité | Description |
| -----------| ---------- |
| Nouvelles mesures d’exportation de Profil | Mesures Ajoutées pour profil des tâches d’exportation montrant le nombre total de profils exportés et le nombre de profils dans chaque espace de nommage. |
| Nouvelles mesures d’statistiques d’observabilité | L’API Observability Insights présente désormais les mesures suivantes pour la diffusion en flux continu de l’assimilation dans [!DNL Profile] : Taux De Demande Entrante, Taux D&#39;Engestion Réussi, Taille D&#39;Enregistrement Ingérée. |
| Point de terminaison de GET en bloc | Point de terminaison de GET en bloc Ajouté à l&#39;API [!DNL Real-time Customer Profile] pour permettre la récupération de plusieurs résultats dans un appel d&#39;API unique. Vous pouvez désormais regrouper jusqu’à 100 ID pour les définitions de segment, les tâches de segment et les stratégies de fusion. |
| Parcourir le profil par identité | Dans l&#39;interface utilisateur [!DNL Platform], vous pouvez désormais sélectionner un espace de nommage d&#39;identité et fournir une valeur d&#39;identité afin de parcourir un profil. |

## Sources {#sources}

Adobe Experience Platform peut ingérer des données à partir de sources externes tout en vous permettant de structurer, d’étiqueter et d’améliorer ces données à l’aide des services [!DNL Platform]. Vous pouvez ingérer des données provenant de diverses sources telles que les applications Adobe, le stockage dans le cloud, les logiciels tiers et le système de gestion de la relation client.

[!DNL Experience Platform] fournit une API RESTful et une interface utilisateur interactive qui vous permet de configurer facilement des connexions source à différents fournisseurs de données. Ces connexions source vous permettent de vous authentifier et de vous connecter à des services de gestion de la relation client et à des systèmes de stockage externes, de définir des heures d’ingestion et de gérer le débit d’ingestion des données.

**Nouvelles fonctionnalités**

| Fonctionnalité | Description |
| ------- | ----------- |
| Prise en charge d’API et d’interface utilisateur supplémentaires pour les systèmes d’enregistrement cloud | Nouveaux connecteurs source pour [!DNL Azure File Storage]. |
| Prise en charge supplémentaire des API et de l’interface utilisateur pour les bases de données | Nouveaux connecteurs source pour [!DNL Azure Data Explorer], IBM DB2 et [!DNL Oracle DB]. |
| Partage de données entre Adobe Audience Manager et [!DNL Experience Platform] | Le processus de mise en service du connecteur [!DNL Audience Manager] a été mis à jour. [!DNL Audience Manager] les jeux de données pour  [!DNL Real-time Customer Profile] sont désormais désactivés par défaut. Vous pouvez choisir manuellement les jeux de données à convertir en [!DNL Profile]. Les nouveaux paramètres par défaut ne sont pas rétroactifs et n&#39;ont qu&#39;un impact sur l&#39;approvisionnement des nouveaux connecteurs [!DNL Audience Manager]. Pour plus d&#39;informations, consultez le [Guide de l&#39;utilisateur des jeux de données](../../catalog/datasets/user-guide.md). |
