## Mises à jour de l’interface utilisateur {#ux}

Adobe Experience Platform publie des mises à jour sur la barre de domaine et d’en-tête afin d’améliorer votre expérience et de réaliser l’union avec d’autres applications [!DNL Experience Cloud].

- Il est plus facile de basculer entre vos organisations ou vers une autre application
- Amélioration de l’aide à l’utilisateur, y compris les articles présentés et la documentation contextuelle dans le menu Aide
- Possibilité de fournir des commentaires sur [!DNL Experience Platform] et les tickets d’assistance pour les fichiers

Le déploiement de la nouvelle expérience est graduel. Vous pouvez afficher l’expérience à l’adresse [https://experience.adobe.com/platform](https://experience.adobe.com/platform).

## [!DNL Data Science Workspace] {#dsw}

[!DNL Data Science Workspace] dans utilise lʼapprentissage automatique et lʼintelligence artificielle pour exploiter les informations contenues dans vos données. Intégré à Adobe Experience Platform, [!DNL Data Science Workspace] vous aide à obtenir des prévisions en utilisant votre contenu et des ressources de données de lʼensemble des solutions Adobe. [!DNL Data Science Workspace] permet d’y parvenir en utilisant [!DNL JupyterLab]. [!DNL JupyterLab] est une interface utilisateur web pour [[!DNL Project Jupyter]](https://jupyter.org/) et est étroitement intégré à Adobe Experience Platform. Il fournit un environnement de développement interactif permettant aux spécialistes des données de travailler avec des notebooks, du code et des données [!DNL Jupyter].

**Nouvelles fonctionnalités**

| Fonctionnalité | Description |
|--- | ---|
| [!DNL JupyterLab Launcher] | [!DNL JupyterLab Launcher] comprend désormais des démarrages pour les notebooks [!DNL Spark] 2.4. [!DNL Spark] Les démarrages de notebook 2.3 sont désormais marqués comme obsolètes et définis pour être supprimés dans une version ultérieure. |
| [!DNL Spark] 2,4 | Les nouvelles recettes Scala ([!DNL Spark]) et PySpark utilisent désormais [!DNL Spark] 2.4. |
| Noyaux | Les notebooks Scala ([!DNL Spark]) sont désormais créés via le noyau Scala. Les notebooks PySpark sont désormais créés via le noyau [!DNL Python]. Le noyau [!DNL Spark] et PySpark sont obsolètes et doivent être supprimés dans une version ultérieure. |
| Recettes | Les nouvelles recettes PySpark et [!DNL Spark] suivent désormais le workflow [!DNL Docker] semblable aux recettes [!DNL Python] et R. |

Pour plus d’informations sur la migration de vos notebooks et de vos recettes pour utiliser [!DNL Spark] 2.4, consultez le [guide d’accès aux données des notebooks](../../data-science-workspace/jupyterlab/access-notebook-data.md). Pour plus d’informations sur Data Science Workspace, consultez la [documentation de présentation](../../data-science-workspace/home.md).

## [!DNL Destinations] {#destinations}

Dans la [plateforme de données clients en temps réel](../../rtcdp/overview.md), les destinations sont des intégrations préconfigurées avec des plateformes de destination qui activent les données vers ces partenaires de manière transparente.

**[!DNL Facebook]**

La plateforme CDP en temps réel prend désormais en charge l’activation des données vers [!DNL Facebook], ce qui vous permet d’activer les profils de vos campagnes [!DNL Facebook] pour le ciblage, la personnalisation et la suppression des audiences en fonction des courriers électroniques hachés.

Pour plus d’informations sur la nouvelle fonctionnalité, voir la page [Destination Facebook](../../destinations/catalog/social/facebook.md) .

**[!DNL Amazon Kinesis]et destinations  [!DNL Azure Event Hubs] de stockage dans le cloud en continu**

Vous trouverez la liste des cas d’utilisation les plus récents dans notre [feuille de route publique](https://github.com/adobe/alloy/projects/5) qui est fréquemment mise à jour.

| Adobe Analytics | Envoyez des données aux suites de rapports [!DNL Analytics] via la configuration Edge. XDM est aplati dans les données contextuelles et prend en charge le balisage multisuite. |
| Adobe Target | Prise en charge d’Adobe Target. Y compris le VEC, le compositeur d’après les formulaires, A/B, XT, Automated Personalization, MVT. |

## [!DNL Real-time Customer Profile] {#profile}

Adobe Experience Platform vous permet d’offrir aux clients des expériences coordonnées, cohérentes et pertinentes, quel que soit l’endroit ou le moment où ils interagissent avec votre marque. Avec [!DNL Real-time Customer Profile], vous pouvez obtenir une vue d’ensemble de chaque client qui combine des données provenant de plusieurs canaux, notamment des données en ligne, hors ligne, CRM et tierces. [!DNL Profile] vous permet de consolider vos données client disparates en une vue unifiée offrant un compte horodaté et exploitable de chaque interaction client.

**Nouvelles fonctionnalités**

| Fonctionnalité | Description |
| -----------| ---------- |
| Nouvelles mesures d’exportation de profils | Ajout de mesures aux tâches d’exportation de profils indiquant le nombre total de profils exportés et le nombre de profils dans chaque espace de noms. |
| Nouvelles mesures Observability Insights | L’API Observability Insights présente désormais les mesures suivantes pour l’ingestion par flux dans [!DNL Profile] : Taux De Requêtes Entrantes, Taux D’Ingestion Réussie, Taille D’Enregistrement Ingérée. |
| Point de terminaison de GET en bloc | Ajout du point de terminaison de GET en bloc à l’API [!DNL Real-time Customer Profile] pour permettre la récupération de plusieurs résultats dans un seul appel API. Vous pouvez désormais regrouper jusqu’à 100 ID pour les définitions de segment, les tâches de segmentation et les stratégies de fusion. |
| Parcourir le profil par identité | Dans l’interface utilisateur [!DNL Platform], vous pouvez désormais sélectionner un espace de noms d’identité et fournir une valeur d’identité afin de parcourir un profil. |

## Sources {#sources}

Adobe Experience Platform peut ingérer des données à partir de sources externes tout en vous permettant de structurer, d’étiqueter et d’améliorer ces données à l’aide des services [!DNL Platform]. Vous pouvez ingérer des données provenant de diverses sources telles que les applications Adobe, le stockage dans le cloud, les logiciels tiers et le système de gestion de la relation client.

[!DNL Experience Platform] fournit une API RESTful et une interface utilisateur interactive qui vous permet de configurer facilement des connexions source à différents fournisseurs de données. Ces connexions source vous permettent de vous authentifier et de vous connecter à des services de gestion de la relation client et à des systèmes de stockage externes, de définir des heures d’ingestion et de gérer le débit d’ingestion des données.

**Nouvelles fonctionnalités**

| Fonctionnalité | Description |
| ------- | ----------- |
| Prise en charge supplémentaire des API et de l’interface utilisateur pour les systèmes de stockage dans le cloud | Nouveaux connecteurs source pour [!DNL Azure File Storage]. |
| Prise en charge supplémentaire des API et de l’interface utilisateur pour les bases de données | Nouveaux connecteurs source pour [!DNL Azure Data Explorer], IBM DB2 et [!DNL Oracle DB]. |
| Partage de données entre Adobe Audience Manager et &lt;a0/[!DNL Experience Platform] | Mise à jour du processus d’approvisionnement du connecteur [!DNL Audience Manager] - [!DNL Audience Manager] les jeux de données pour  [!DNL Real-time Customer Profile] sont désormais désactivés par défaut. Vous pouvez choisir manuellement les jeux de données à convertir en [!DNL Profile]. Les nouveaux paramètres par défaut ne sont pas rétroactifs et n’ont un impact que sur la configuration des nouveaux connecteurs [!DNL Audience Manager]. Pour plus d’informations, reportez-vous au [Guide de l’utilisateur des jeux de données](../../catalog/datasets/user-guide.md). |
