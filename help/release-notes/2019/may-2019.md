## Incorporation de données {#ingestion}

Adobe Experience Platform fournit un ensemble riche de fonctionnalités permettant d’assimiler tout type et toute latence de données. Vous pouvez ingérer à l’aide d’API de traitement par lot ou de diffusion en continu, à l’aide de connecteurs intégrés d’Adobe, de partenaires d’intégration de données ou de l’interface utilisateur d’Adobe Experience Platform.

Pour obtenir de la documentation sur le produit, reportez-vous à la section Présentation [de](../../ingestion/home.md)Data Ingestion.

### Importation par lot

Utilisez l’API d’importation par lot pour assimiler n’importe quel type de données dans Adobe Experience Platform. L&#39;API d&#39;importation en masse fournit un mécanisme sécurisé et évolutif permettant d&#39;assimiler des données conformes aux schémas XDM ou non XDM.

| Fonction | Description |
| -----------| ---------- |
| Prise en charge des formats enrichis | Prend en charge l’assimilation de JSON, Parquet, CSV |
| Validation des données | Trois niveaux de validation sont effectués sur les données : Schéma, type de données et contraintes |
| Chiffrement des données en mouvement | Protège et sécurise les données lors de leur transmission sur les réseaux de données. |
| Téléchargements en plusieurs parties | Téléchargements de fichiers volumineux pris en charge pour des fichiers de 256 Mo ou plus |
| Partitionné automatiquement | Les données sont partitionnées automatiquement afin d’améliorer l’évolutivité et d’optimiser les performances des jeux de données volumineux. |

**Problèmes connus**

Aucun.

Pour obtenir la documentation du produit, consultez la présentation [de l’importation par](../../ingestion/batch-ingestion/overview.md)lot.

Les connecteurs de plateformes vous permettent de vous authentifier sur les systèmes d’enregistrement et les services de gestion de la relation client basés sur le cloud, de définir les heures d’exécution pour l’assimilation et de gérer le débit d’assimilation des données.

| Fonction | Description |
| -----------| ---------- |
| Connecteurs d’Enregistrement basés sur le cloud | Envoi de données depuis Microsoft Azure Blob et l’enregistrement cloud Amazon S3 vers Adobe Experience Platform |
| Microsoft Dynamics CRM &amp; Salesforce CRM | Découvrez toutes les données disponibles dans Microsoft Dynamics ou Salesforce CRM. Invitez n&#39;importe quel objet, standard ou personnalisé de Microsoft Dynamics ou de Salesforce CRM. Les données incrémentielles peuvent être ingérées toutes les heures ou tous les jours. |  |

## Experience Data Model (XDM) {#xdm}

La normalisation et l’interopérabilité sont des concepts clés de la plate-forme d’expérience. Le modèle de données d’expérience (XDM), piloté par Adobe, vise à normaliser les données d’expérience client et à définir des schémas pour la gestion de l’expérience client.

XDM est une spécification documentée publiquement conçue pour améliorer la puissance des expériences numériques. Il fournit des structures et des définitions communes à toute application pour communiquer avec les services d’Adobe Experience Platform. En respectant les normes XDM, toutes les données d’expérience client peuvent être intégrées dans une représentation commune offrant des informations d’une manière plus rapide et plus intégrée. Vous pouvez obtenir des informations précieuses sur les actions client, définir des audiences client par le biais de segments et utiliser les attributs client à des fins de personnalisation.

XDM est le mécanisme qui permet à Experience Cloud, optimisé par Adobe Experience Platform, de transmettre le message approprié à la bonne personne, au bon canal, exactement au bon moment.

Méthodologie sur laquelle la plateforme Experience Platform est créée, XDM System rend opérationnels les schémas de modèles de données d’expérience pour les utiliser par les composants de la plate-forme Experience.

| Fonction | Description |
| -----------| ---------- |
| Registre des Schémas | Le registre des Schémas fournit une interface utilisateur (éditeur de Schémas) et une API RESTful à partir de laquelle toutes les ressources liées au schéma peuvent être consultées et gérées. L&#39;éditeur de Schéma et l&#39;API de registre de Schémas peuvent également être utilisés pour créer et gérer de nouveaux schémas et ressources propres à chaque organisation IMS. |
| Bibliothèque de Schémas | La bibliothèque de Schémas contient toutes les ressources du registre des Schémas, y compris celles définies par une organisation IMS individuelle et celles mises à la disposition de tous les utilisateurs par Adobe, les partenaires Experience Platform et les fournisseurs dont les applications sont intégrées à Experience Platform. |

**Problèmes connus**

Aucun.

Pour en savoir plus sur l&#39;utilisation de XDM à l&#39;aide de l&#39;API Schéma Registry et de l&#39;éditeur de Schémas, consultez la documentation [du système](../../xdm/home.md)XDM.

## Profil client en temps réel {#profile}

Le Profil est un magasin d’entités de recherche générique qui fusionne les données entre différentes ressources de données d’entreprise et fournit un accès à ces données sous la forme de profils de consommation en temps réel et de événements de séries chronologiques connexes, appelés vue d’union. La vue unifiée permet d’informer et d’autonomiser les actions entre les canaux, les plateformes et les intégrations de solutions Adobe. En outre, le Profil est utilisé pour alimenter l&#39;apprentissage automatique et Sensei. Les API de Profil peuvent également être utilisées directement pour enrichir les fonctionnalités des solutions tierces, des systèmes de gestion de la relation client et des solutions propriétaires.

| Fonction | Description |
| -----------| ---------- |
| profil client en temps réel | Le Profil ingère des données par le biais d’entrées de données, les fusionne avec les données existantes dans le magasin de profils et crée ou modifie la vue unifiée en temps quasi réel. Ces données, qui évoluent en temps avec les expériences de vos clients, peuvent être utilisées pour formuler des recommandations personnalisées. |
| vue Union | L&#39;une des principales caractéristiques du Profil est la possibilité d&#39;unifier les données à canaux multiples. Le Profil peut fournir une vue fusionnée des individus dans votre base de consommateurs, entre les ensembles de données et pour les identités liées, appelée vue d&#39;union. |
| Edge | Le service Edge sert de cadre pour la collecte de données à faible latence, l&#39;informatique enfichable et l&#39;activation rapide des données sur tous les canaux adressables. Le service Edge facilite la personnalisation en temps réel. |

**Problèmes connus**

## Identity Service {#identity}

Identity Service résout le problème fondamental posé par les identités déconnectées de vos clients, en raison de la nature fragmentée de leurs données sur chacun des systèmes disparates en jeu dans l&#39;écosystème de votre entreprise. Pour ce faire, il établit un lien entre les identités des différents périphériques et des différents systèmes utilisés par vos clients pour interagir avec votre marque. De cette façon, Identity Service facilite une compréhension complète de vos clients et de leur comportement afin que vous puissiez diffuser des expériences numériques impactées en temps réel.

| Fonction | Description |
| -----------| ---------- |
| Graphique d&#39;identité | Un graphique d&#39;identité est une carte des relations entre les identités qui se met à jour avec l&#39;activité du client en temps quasi réel. Identity Service gère un graphique d&#39;identité visible uniquement par votre entreprise et créé en fonction de vos données, appelé graphique privé. Identity Service augmente votre graphique privé lorsqu&#39;un enregistrement de données assimilées contient plusieurs identités, ajoutant ainsi des relations entre ces identités. |

**Problèmes connus**

Aucun.

Pour obtenir de la documentation sur le produit, voir Présentation [du service](../../identity-service/home.md)d&#39;identité.

## Service de segmentation {#segmentation}

Le service de segmentation définit un sous-ensemble particulier de profils de votre magasin de profils, décrivant les critères qui distinguent un groupe de personnes commercialisables dans votre magasin de profils. Les segments peuvent être basés sur des données d’enregistrement (telles que des informations démographiques) ou des événements de séries chronologiques représentant les points de contact clients avec votre marque.

Par exemple, dans une campagne par courriel intitulée &quot;Avez-vous oublié d’acheter vos chaussures de sport ?&quot;, vous souhaitez peut-être un segment d’audience pour tous les utilisateurs qui ont recherché des chaussures de sport au cours des 30 derniers jours, mais qui n’ont pas effectué d’achat. Vous pouvez également souhaiter cible le contenu Web afin qu’il s’affiche uniquement pour les utilisateurs dont le programme de récompenses comporte une certaine évaluation.

| Fonction | Description |
| -----------| ---------- |
| API de segmentation | Les services RESTful vous permettent d’accéder par programmation à la création de segments de clients. |
| Créateur de segments | Le créateur de segments est un espace de travail robuste et intuitif qui permet de créer des segments de clients à partir de vos données de Profil. |

**Problèmes connus**

Aucun.

Pour obtenir de la documentation sur le produit, reportez-vous à la section Présentation du service [de segmentation.](../../segmentation/home.md)

## Gouvernance des données {#governance}

Adobe Experience Platform permet aux sociétés de rassembler les données issues de plusieurs systèmes d’entreprise afin de mieux permettre aux spécialistes du marketing d’identifier, de comprendre et d’impliquer les clients. La plate-forme d’expérience comprend une infrastructure de gouvernance des données de bout en bout, y compris l’étiquetage et l’application de l’utilisation des données (DULE), afin d’assurer l’utilisation appropriée des données dans la plate-forme et lors du partage entre les systèmes.

La gouvernance des données est une série de stratégies et de technologies utilisées pour gérer les données des clients et assurer la conformité aux réglementations, restrictions et politiques applicables à l&#39;utilisation des données. Il joue un rôle clé dans la plate-forme d’expérience à divers niveaux, notamment le catalogage, le lignage des données, l’étiquetage de l’utilisation des données, les stratégies d’accès aux données et le contrôle d&#39;accès sur les données pour les actions marketing.

La prise en main de la gouvernance des données nécessite une compréhension approfondie des réglementations, obligations contractuelles et stratégies d’entreprise qui s’appliquent à vos données client. À partir de là, les données peuvent être classifiées en appliquant les étiquettes d&#39;utilisation appropriées et leur utilisation peut être contrôlée par la définition de stratégies d&#39;utilisation des données.

La structure DULE simplifie et rationalise le processus de catégorisation des données et de création de stratégies d’utilisation des données par le biais de l’interface utilisateur de la plateforme d’expérience et de l’API DULE Policy Service.

| Fonction | Description |
| -----------| ---------- |
| Étiqueter les entités de plateforme d’expérience | Appliquez et gérez des étiquettes d’utilisation des données au niveau des champs de connexion, de jeu de données et de jeu de données en fonction d’étiquettes catégorisées prédéfinies. |
| Création de stratégies d’utilisation des données | Définissez et gérez des stratégies d’utilisation des données en fonction de la présence d’étiquettes et des actions marketing demandées. |
| Vérifier les violations de stratégie | Déterminer si les stratégies d’utilisation définies sont violées lors de l’exécution d’actions marketing sur les données de la plate-forme. |

**Problèmes connus**

* Aucun.

Pour plus d&#39;informations, consultez la présentation [de la gouvernance des](../../data-governance/home.md)données).
